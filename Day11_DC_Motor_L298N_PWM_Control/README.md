## 系统分工

| 部件 | 作用 |
|---|---|
| STM32 | 读取按键并输出 PWM 控制信号 |
| L298N | 根据输入信号切换 H 桥，为电机提供驱动通路 |
| 电池 | 提供电机运行所需能量 |
| 直流有刷电机 | 将电能转换为机械运动 |

L298N 本身不产生能量。STM32 负责控制，L298N 负责功率驱动和电流方向切换，电池才是电机能量来源。

## 控制链

```text
按键
-> KEY_Scan()
-> 修改 TIM1 的 CCR
-> TIM1 输出 PWM
-> PA8、PA9
-> L298N IN1、IN2
-> H 桥
-> 改变电机两端电压方向和有效驱动时间
-> 控制转向与转速
```

本工程没有独立的 `motor.c` 或 `l298n.c`，电机方向控制逻辑直接写在 `main.c` 中。

## 为什么不能直接使用 GPIO 驱动电机

- STM32 GPIO 的输出电流有限，不能满足电机启动和运行所需电流。
- 电机启动和堵转电流通常远大于正常运行电流。
- 电机是感性负载，换向和断电时可能产生反电动势。
- 电刷和绕组会带来电磁干扰，可能造成 STM32 复位或运行异常。
- 应使用 L298N 等专用驱动电路分隔控制部分与功率部分。

GPIO 只向 L298N 提供逻辑控制信号，不能直接接到电机两端。

## L298N 与 H 桥

H 桥通过切换不同开关组合改变流过电机绕组的电流方向：

```text
方向 A：电流从电机一端流向另一端
方向 B：电流反向流动
```

电流方向改变后，电磁转矩方向也随之改变，因此电机反向旋转。

IN1、IN2 是控制端，不是电机能量输入端。“正转”和“反转”只是人为定义，实际方向取决于 OUT1、OUT2 与电机两根线的连接顺序。

## 实际引脚与通道

根据 `pwm.c` 的实际配置：

| STM32 引脚 | TIM1 通道 | 本实验用途 |
|---|---|---|
| PA8 | TIM1_CH1 | 预期连接 L298N IN1 |
| PA9 | TIM1_CH2 | 预期连接 L298N IN2 |
| PA10 | TIM1_CH3 | 已初始化，但主程序未使用 |
| PA11 | TIM1_CH4 | 已初始化，但主程序未使用 |

源码中没有定义名为 `IN1`、`IN2` 的宏，PA8/PA9 与 L298N 输入端的对应关系依赖实际接线。按键连接为：

| 按键 | STM32 引脚 | 输入模式 |
|---|---|---|
| KEY1 | PB4 | 上拉输入 |
| KEY2 | PB5 | 上拉输入 |

## IN1、IN2 基本状态

在 L298N 对应桥臂已使能的常见情况下：

| IN1 | IN2 | 一般状态 |
|---:|---:|---|
| 0 | 0 | 停止或动态制动 |
| 0 | 1 | 向一个方向转动 |
| 1 | 0 | 向相反方向转动 |
| 1 | 1 | 停止或动态制动 |

具体停止效果还取决于模块结构、ENA 状态和接线。ENA 关闭时通常是高阻滑行；桥臂使能且两输入同电平时通常表现为制动，不能把所有“IN1 等于 IN2”的情况简单视为完全相同。

## 原工程的实际控制关系

主程序初始化：

```c
TIM1_PWM_Init(3000 - 1, 0);
```

`pwm.c` 将各通道初始脉冲值设置为 0，因此上电初始化后 CH1、CH2 都为 0。

按键对应关系：

| 状态 | CCR1 / PA8 | CCR2 / PA9 | 预期效果 |
|---|---:|---:|---|
| 初始化 | 0 | 0 | 停止或制动，取决于 L298N 使能状态 |
| KEY1 | 1500 | 0 | 约 50% PWM，方向 A |
| KEY2 | 0 | 1500 | 约 50% PWM，方向 B |

KEY1 代码：

```c
TIM_SetCompare1(TIM1, 1500);
TIM_SetCompare2(TIM1, 0);
```

KEY2 代码：

```c
TIM_SetCompare1(TIM1, 0);
TIM_SetCompare2(TIM1, 1500);
```

`KEY_Scan(0)` 使用单次按键模式，并包含 10 ms 消抖。若两个按键同时按下，代码优先返回 KEY1。

## PWM 参数

实际初始化参数：

```c
TIM1_PWM_Init(3000 - 1, 0);
```

| 参数 | 实际值 | 含义 |
|---|---:|---|
| TIM1 时钟 | 72 MHz | 计算采用的定时器时钟 |
| PSC | 0 | 实际分频系数为 `PSC + 1 = 1` |
| ARR | 2999 | 实际计数次数为 `ARR + 1 = 3000` |

PWM 频率：

```text
f_PWM = 72 MHz / ((PSC + 1) * (ARR + 1))
      = 72000000 / (1 * 3000)
      = 24000 Hz
      = 24 kHz
```

占空比近似为：

```text
D = CCR / (ARR + 1)
```

| CCR | 近似占空比 |
|---:|---:|
| 750 | 25% |
| 1500 | 50% |
| 2250 | 75% |

本工程实际按键控制使用的是 CCR 1500，即约 50% 占空比。

## PWM 调速原理

- 修改 CCR 改变的是 PWM 占空比，不是 PWM 频率。
- 高电平电压幅值基本不变，变化的是一个周期中的有效驱动时间。
- 占空比增大时，电机获得的等效平均电压和平均功率通常提高。
- 实际转速还受负载、摩擦、电源内阻、驱动压降和电机特性影响，不会严格与占空比成正比。
- 占空比过低时，电机可能因为静摩擦和启动转矩不足而无法启动。

## 共地

STM32 GND、L298N GND 和电池负极需要连接在一起，建立统一的 0 V 参考。否则 L298N 无法可靠判断 STM32 输出的高低电平。

共地不表示电机大电流流入 STM32。电机主电流主要沿下面的功率回路流动：

```text
电池正极 -> L298N -> 电机 -> L298N -> 电池负极
```

STM32 主要提供控制信号。

## 停止与制动

### 滑行停止

驱动桥输出呈高阻，电机失去驱动力后依靠负载和机械摩擦逐渐停下。

### 动态制动

电机两端被置于相同电位或形成低阻回路。电机因惯性继续旋转时产生的电流会形成反向转矩，使其更快停止。

实际是滑行还是制动，与 L298N 的 IN1、IN2、ENA 状态以及模块接法有关。

## 当前代码的换向风险

当前代码存在初始化停止状态，但没有独立的停止按键或运行中的停止分支。电机一旦由某个按键启动，会保持最后一次设置的 CCR，直到按下另一个方向键或系统复位。

当 KEY1 和 KEY2 依次按下时，程序会直接把：

```text
CCR1=1500, CCR2=0
```

切换为：

```text
CCR1=0, CCR2=1500
```

代码没有软启动、减速、明确的 `CCR1=0/CCR2=0` 停顿时间或等待电机停稳的逻辑。因此从控制意图看，它允许直接正反转切换。即使 CCR 使用了预装载，也没有人为设计的安全停顿。

更可靠的换向过程应为：

```text
逐步降低占空比
-> 两路输出进入停止状态
-> 等待电机速度下降
-> 设置相反方向
-> 逐步提高占空比
```

本仓库只记录这一风险，没有修改原始代码或添加保护逻辑。

## 安全问题

- 必须断电接线。
- 通电前确认电池正负极和 DC5521 转接端子的极性。
- STM32、L298N 和电池必须共地。
- 不能把电机直接连接到 STM32 GPIO。
- 未使用的编码器线应分别绝缘，不能相互短接。
- 电机堵转时电流很大，应立即断电。
- 高速正转时不要立即切换为高速反转。
- 换向应先减速、停止，再反向加速。
- L298N 明显发热时应检查负载、堵转、短路、供电和散热。
- L298N 本身存在压降和功耗，长时间大电流运行时需注意温升。
- 额定 8 V 电机使用 12 V 电源时，`8 / 12 = 66.7%` 只能作为平均电压的粗略估算，不能保证绝对安全。
- PWM 导通时电机端仍可能承受接近电源电压的瞬时脉冲，启动、堵转和换向电流也不能只用平均电压判断。

## 今日容易混淆的地方

- IN1 高电平只是控制信号，不表示电机电流从 IN1 流入。
- 两路 100% 不代表速度叠加，电机关注的是两端电压差。
- CCR 改变占空比，不改变由 PSC 和 ARR 决定的 PWM 频率。
- L298N 不产生能量，电池才是能量来源。
- 共地是统一电压参考，不是让电机大电流经过 STM32。
- “正转”和“反转”会随电机接线交换而改变。
- `0/0`、`1/1` 和 ENA 关闭的停止效果可能不同。
