## 为什么需要输出变化率限制

PID 根据当前误差计算控制量。当误差突然增大或负载突然消失时，理想输出可能在一个控制周期内发生很大的跳变。

如果 PWM 上升过快，电机启动会比较突兀；如果卸载后 PWM 下降过慢，电机会在一段时间内继续维持过高驱动力，从而产生较明显的速度超调。

因此，在 PID 理想输出和实际 PWM 输出之间加入变化率限制：

```text
PID 计算理想输出
  -> 限制本周期允许增加或减少的最大幅度
  -> 执行最终 0～1000 限幅
  -> 得到实际 PWM 输出 a_m
```

## `new_output` 的含义

`off_cal` 是增量式 PID 本次希望施加的输出变化量，因此：

```c
new_output = a_m + off_cal;
```

`new_output` 表示 PID 根据当前误差计算出的候选输出，也就是还没有经过输出变化率限制和最终范围限幅的理想结果。

它是一个局部变量，只服务于当前一次 `pid_control()` 调用。经过变化率限制后，`new_output` 才会赋给全局输出 `a_m`。

## PID 理想输出和实际 PWM 输出

两者的区别如下：

| 名称 | 代码中的表示 | 含义 |
|---|---|---|
| PID 增量 | `off_cal` | 本周期由 P、I、D 计算出的期望变化量 |
| PID 理想输出 | `new_output` | `a_m + off_cal`，尚未限制变化速度 |
| 实际 PWM 输出 | `a_m` | 经过变化率限制和 `0～1000` 限幅后的输出 |

变化率限制不会改变 PID 的误差计算方式，而是在理想控制量到执行器输出之间增加一道约束。

## 非对称输出变化率限制

当前使用的限制为：

```text
上升：每个控制周期最多 +30
下降：每个控制周期最多 -200
```

判断逻辑为：

```c
if (new_output - a_m > 30.0f)
{
    new_output = a_m + 30.0f;
}
else if (new_output - a_m < -200.0f)
{
    new_output = a_m - 200.0f;
}
```

## 为什么上升和下降不使用同一个限制

上升和下降面对的是两个不同目标：

- PWM 上升需要适当放慢，使电机启动更平滑，减小机械冲击和电流突变。
- PWM 下降需要足够快，使负载突然消失后驱动力及时减小，避免高 PWM 状态持续过久。

如果上升和下降都限制为 30，卸载后 PWM 回落会过慢；如果两边都使用 200，启动时输出又可能增长过快。因此本次采用较慢上升、较快下降的非对称限制。

## 当前完整 `pid_control()` 核心代码

> 当前工程中已经验证的微分项使用 `+ D * (...)`。本次整理保留实际符号，不改变现有控制逻辑。

```c
float pid_control(float speed)
{
    float new_output;

    err_last_last = err_last;
    err_last = err;
    err = aim_v - speed;

    off_cal = P * (err - err_last)
            + D * (err - 2.0f * err_last + err_last_last);

    /* Stop integration only when it would push the output further into saturation. */
    if (!((a_m >= 1000.0f && err > 0.0f) ||
          (a_m <= 0.0f && err < 0.0f)))
    {
        off_cal += I * err;
    }

    /* Ideal PID output before slew-rate limiting. */
    new_output = a_m + off_cal;

    /* Limit each control period to +30 when rising and -200 when falling. */
    if (new_output - a_m > 30.0f)
    {
        new_output = a_m + 30.0f;
    }
    else if (new_output - a_m < -200.0f)
    {
        new_output = a_m - 200.0f;
    }

    a_m = new_output;

    /* Clamp the final PWM output. */
    if (a_m > 1000.0f)
        a_m = 1000.0f;
    else if (a_m < 0.0f)
        a_m = 0.0f;

    return a_m;
}
```

## 第一次实验：上升 `+30`，下降 `-100`

第一组实验将输出变化限制为：

```text
每周期最多增加 30
每周期最多减少 100
```

实验结果：

- 电机启动到目标速度约需 6 秒
- 稳定 PWM 约为 737
- 突然卸载后的最高速度约为 `0.88 r/s`
- 恢复到约 `0.50 r/s` 需要约 6～7 秒

启动过程较平滑，但下降限制只有 100，使卸载后 PWM 回落偏慢，高输出状态持续时间较长，超调和恢复时间仍然比较明显。

## 第二次实验：上升 `+30`，下降 `-200`

第二组实验保持上升限制不变，只把每周期最大下降量从 100 提高到 200：

```text
每周期最多增加 30
每周期最多减少 200
```

实验结果：

- 启动时间仍约为 6 秒
- 突然卸载后的最高速度下降到 `0.7x r/s`
- 卸载超调明显改善

上升限制没有改变，因此启动时间基本不变；下降速度加快后，卸载时 PWM 能更快脱离高输出区，速度峰值随之降低。

## 两组实验对比

| 项目 | 实验一：`+30/-100` | 实验二：`+30/-200` |
|---|---:|---:|
| 启动时间 | 约 6 秒 | 约 6 秒 |
| 稳定 PWM | 约 737 | 当前实验重点未重新记录 |
| 卸载最高速度 | 约 `0.88 r/s` | 约 `0.7x r/s` |
| 恢复到 `0.50 r/s` | 约 6～7 秒 | 明显加快 |
| 卸载超调 | 较明显 | 明显改善 |

## 实验结论

- 上升限制主要影响启动平滑性和启动时间。
- 下降限制主要影响突然卸载后的速度超调和恢复时间。
- 下降过慢会让高 PWM 状态持续过久，即使已经加入抗积分饱和，仍可能产生较大超调。
- 在当前电机、负载和控制周期下，`+30/-200` 比 `+30/-100` 更适合当前系统。
- 输出变化率限制与抗积分饱和解决的是不同问题，两者可以同时保留。
