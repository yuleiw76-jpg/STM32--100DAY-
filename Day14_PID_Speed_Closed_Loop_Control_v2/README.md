# 一、闭环控制结构

本实验的闭环控制流程如下：

```text
目标速度 aim_v
        ↓
计算误差 error
        ↓
增量式 PID
        ↓
得到 PWM 输出
        ↓
TIM1 输出 PWM
        ↓
L298N 驱动电机
        ↓
编码器测量实际速度
        ↓
speed 反馈给 PID
        ↓
继续修正 PWM
```

与固定 PWM 的开环控制不同，闭环控制会持续测量实际速度，并根据误差自动修正输出。

---

# 二、P、I、D 三项的作用

## 1. 比例项 P

比例项关注当前误差。

```text
误差越大，修正力度越大
误差越小，修正力度越小
```

P 太小：

- 响应速度慢
- 输出可能不足以启动电机

P 太大：

- 容易超调
- 容易产生震荡

---

## 2. 积分项 I

积分项关注误差持续存在的时间。

在本项目的增量式 PID 中，积分部分为：

```c
I * err
```

只要误差持续存在，每次 PID 计算都会继续改变总输出。

例如电机静止时：

```text
目标速度 = 0.50 r/s
实际速度 = 0
误差 = 0.50
I = 30
```

每次 PID 计算产生的积分增量为：

```text
30 × 0.50 = 15
```

因此 PWM 会逐渐增加：

```text
50 → 65 → 80 → 95 → ...
```

直到 PWM 足够大，使电机启动。

积分项的主要作用是：

> 消除长期存在的稳态误差。

但积分过大或误差持续时间过长，也可能造成积分累积，使负载突然消失后产生超调。

---

## 3. 微分项 D

微分项关注误差变化的速度。

它的作用类似于提前刹车：

- 当速度快速接近目标时，提前减小输出
- 抑制超调
- 减少震荡
- 提高系统稳定性

但 D 对噪声比较敏感。

如果编码器测速出现瞬时抖动，D 可能放大这种快速变化，使 PWM 输出出现波动。

---

# 三、位置式 PID 与增量式 PID

## 位置式 PID

位置式 PID 直接计算当前完整输出：

```text
u(k)
```

计算出的结果可以直接作为 PWM 或 CCR 的值。

可以理解为：

> 当前输出应该是多少。

---

## 增量式 PID

增量式 PID 先计算本次输出应该改变多少：

```text
Δu(k)
```

然后再加到上一次输出上：

```c
a_m += off_cal;
```

可以理解为：

> 当前输出应该在上一次基础上增加或减少多少。

---

# 四、增量式 PID 代码分析

本项目修改后的增量式 PID 公式为：

```c
off_cal = P * (err - err_last)
        + I * err
        + D * (err - 2 * err_last + err_last_last);
```

其中：

```text
err                  当前误差
err_last             上一次误差
err_last_last        上上次误差
off_cal              本次输出增量
a_m                  累计后的总输出
```

随后执行：

```c
a_m += off_cal;
```

所以：

- `off_cal` 是当前控制周期算出的 PWM 增量
- `a_m` 是实际写入 PWM 比较寄存器的总输出

---

## 误差更新

```c
err_last_last = err_last;
err_last = err;
err = aim_v - speed;
```

更新后：

```text
err            = e(k)
err_last       = e(k-1)
err_last_last  = e(k-2)
```

---

# 五、PWM 输出与方向控制

PID 函数返回当前总输出：

```c
out_duty = pid_control(speed);
```

当输出为正时：

```c
TIM_SetCompare1(TIM1, out_duty);
TIM_SetCompare2(TIM1, 0);
```

当输出为负时：

```c
TIM_SetCompare1(TIM1, 0);
TIM_SetCompare2(TIM1, -out_duty);
```

因此：

```text
out_duty 的绝对值 → PWM 强度
out_duty 的正负号 → 电机方向
```

`TIM_SetCompare1()` 本质上是修改：

```c
TIM1->CCR1
```

`TIM_SetCompare2()` 本质上是修改：

```c
TIM1->CCR2
```

# 六、编码器测速

编码器使用 TIM4 编码器模式读取 A、B 两相信号：

```text
A 相 → PB6
B 相 → PB7
```

理论上输出轴转一圈产生：

```text
13 × 30 × 4 = 1560 个计数
```

实测结果：

```text
初始计数：1560
转动圈数：5 圈
结束计数：-6274
```

计算：

```text
计数差 = |-6274 - 1560|
       = 7834

每圈计数 = 7834 ÷ 5
         ≈ 1566.8
```

实测值与理论值 1560 非常接近，误差主要来自手动转动时起点和终点无法完全重合。

因此本项目继续使用：

```text
1560 counts/revolution
```

---

## 测速公式

TIM3 每 100 ms 计算一次速度：

```c
speed = (float)(
    (circle_count - last_count) * 1560
    + (int)TIM4->CNT
    - last_num
) / 1560.0f / 0.1f;
```

计算过程：

```text
0.1 秒内计数差
        ↓
除以 1560
        ↓
得到 0.1 秒内转过的圈数
        ↓
除以 0.1 秒
        ↓
得到 r/s
```

由于编码器计数方向与电机正向定义相反，本次实验增加了：

```c
speed = -speed;
```

使正向 PWM 对应正速度反馈。

# 七、定时器分工

本项目中多个定时器分别负责不同任务。

## TIM1

负责输出 PWM，控制 L298N 和电机。

```text
ARR = 2999
PWM 频率约为 24 kHz
```

---

## TIM4

工作在编码器模式，读取 PB6、PB7 上的 A、B 两相信号。

---

## TIM3

每 100 ms 进入一次中断，用于计算电机实际速度。

---

## TIM2

每 100 ms 进入一次中断，用于执行 PID 计算并更新 PWM。

完整周期：

```text
TIM3 测速
→ 更新 speed
→ TIM2 调用 PID
→ 更新 CCR
→ 电机速度改变
→ 编码器重新反馈
```

# 九、积分累积现象

当用手按住输出轴时，速度下降：

```text
实际速度下降
→ 误差保持为正
→ I × error 持续加入输出
→ a_m 不断增加
```

按住时间越长，累计的 PWM 越大。

突然松开后：

```text
负载瞬间消失
→ 已累计的 PWM 仍然较大
→ 电机短暂冲过目标速度
```

因此观察到：

> 按住输出轴越久，松手后的最大速度越高。

这是典型的积分累积现象。

合适的 D 可以在速度快速上升时提前削弱输出，但不能完全消除长时间积分累积造成的影响。

工程中通常还会使用：

- 积分限幅
- 输出饱和时停止积分
- 条件积分
- 抗积分饱和

# 十、今日故障排查记录

今天实验中出现了多处问题。

## 1. 编码器 GND 未连接

编码器与 STM32 没有公共参考电位，A、B 信号无法被可靠识别。

---

## 2. 编码器 5V 接到了 GND

编码器没有获得正确供电，A、B 始终无法输出正常电平，并存在短路风险。

---

## 3. PB6、PB7 使用浮空输入

原配置：

```c
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IN_FLOATING;
```

电机静止时，计数仍然会自动变化，说明噪声被误识别为脉冲。

修改为：

```c
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;
```

使用内部上拉后，静止计数恢复稳定。

---

## 4. 编码器 A 相接线错误

导致 TIM4 无法收到正确的正交信号，转动输出轴时计数器不变化。

---

## 5. 反馈方向相反

固定正向 PWM 时，速度最初显示：

```text
-0.60 r/s
```

而目标速度为正数。

如果直接进入 PID：

```text
error = target - negative_speed
```

控制器会认为误差很大，继续增加输出，可能使负反馈变成错误反馈。

解决方法：

```c
speed = -speed;
```

---

## 6. TIM2 中断函数被注释

虽然 TIM2 初始化已经恢复，但中断函数仍被注释，因此 PID 函数完全没有运行。

表现为：

- 电机不转
- `now` 一直为 0
- 即使增加 I 参数也没有变化

恢复 `TIM2_IRQHandler()` 后，PID 控制正常执行。

---

## 7. 原 PID 公式参数位置错位

原代码中 P、I、D 对应的数学项位置不正确。

修改为标准增量式 PID 结构：

```c
off_cal = P * (err - err_last)
        + I * err
        + D * (err - 2 * err_last + err_last_last);
```

# 十一、今日调试方法总结

今天使用了逐层隔离的调试方法：

```text
先关闭 PID
→ 固定 PWM
→ 检查电机是否稳定单方向转动
→ 检查 GPIO 原始电平
→ 检查 TIM4->CNT
→ 检查总计数
→ 实测每圈计数
→ 检查测速符号
→ 恢复 PID
→ 调整 P、I、D
```

这个过程说明：

> 出现闭环失控时，不能直接只调 PID 参数，必须先确认传感器反馈、方向、供电和底层计数全部正确。

---

# 十二、今日最大的收获

今天真正理解了：

1. PID 的输入必须是可靠的真实反馈。
2. 反馈方向错误时，闭环可能比开环更加危险。
3. 增量式 P 只响应误差变化，固定误差不会让 P 持续增加输出。
4. I 会让持续存在的误差不断改变总输出。
5. D 关注误差变化速度，可以提前抑制超调。
6. 编码器信号不仅需要 A、B，还必须正确供电并共地。
7. 调试复杂系统时，应从原始信号逐层向上排查。
