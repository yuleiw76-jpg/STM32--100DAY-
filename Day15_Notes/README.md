## 积分饱和的含义

当电机受到阻力时，实际速度会低于目标速度：

```text
speed < aim_v
```

此时误差 `err = aim_v - speed` 为正，积分项会持续推动 PID 输出增大。但是本工程中的 PWM 输出存在范围限制：

```text
0 <= a_m <= 1000
```

当 `a_m` 已经达到上限 1000 后，继续增大控制量也无法让驱动器输出更高的 PWM。如果积分仍沿着相同方向不断累积，那么负载突然消失后，控制器需要先消耗掉多余的积分量，输出才能下降。这会导致电机在卸载后出现更明显、持续更久的速度超调。

这种“执行器已经饱和，但积分仍继续累积”的现象就是积分饱和，也常称为积分累积或 Integral Windup。

## 条件积分抗饱和的原理

本次实验采用条件积分法：先计算比例项和微分项，再判断当前积分项是否会把输出继续推向饱和区。

- 当 `a_m >= 1000` 且 `err > 0` 时，误差还想让输出继续增大，因此暂停积分。
- 当 `a_m <= 0` 且 `err < 0` 时，误差还想让输出继续减小，因此暂停积分。
- 当误差方向有助于输出离开饱和区时，允许积分恢复。
- 当输出未处于上下限时，积分正常参与计算。

核心判断条件为：

```c
if (!((a_m >= 1000.0f && err > 0.0f) ||
      (a_m <= 0.0f && err < 0.0f)))
{
    off_cal += I * err;
}
```

这样做并不是取消积分，而是只在积分会让饱和进一步恶化时暂时停止积分。

## 修改后的 `pid_control()` 核心代码

```c
float pid_control(float speed)
{
    err_last_last = err_last;
    err_last = err;
    err = aim_v - speed;

    off_cal = P * (err - err_last)
            + D * (err - 2.0f * err_last + err_last_last);

    if (!((a_m >= 1000.0f && err > 0.0f) ||
          (a_m <= 0.0f && err < 0.0f)))
    {
        off_cal += I * err;
    }

    a_m += off_cal;

    if (a_m > 1000.0f)
        a_m = 1000.0f;
    else if (a_m < 0.0f)
        a_m = 0.0f;

    return a_m;
}
```

## 当前 PID 参数

```c
P = 100;
I = 30;
D = 5;
aim_v = 0.5;
```

目标速度 `aim_v = 0.5`，单位为 `r/s`。

## `a_m` 的跨文件使用

`a_m` 是 PID 控制器计算得到的 PWM 输出。它只在一个源文件中定义，再通过头文件向其他源文件声明。

### `pid.c`：定义变量

```c
float a_m = 0.0f;
```

这是真正的变量定义，会为 `a_m` 分配存储空间。

### `pid.h`：使用 `extern` 声明

```c
extern float a_m;
```

`extern` 不会创建另一份变量，只是告诉包含该头文件的源文件：`a_m` 已经在其他位置定义。

### `main.c`：包含头文件后使用

```c
#include "pid.h"
```

包含 `pid.h` 后，`main.c` 可以读取同一份 `a_m`，无需也不应该再次定义它。

数据关系如下：

```text
pid.c：定义并更新 a_m
  -> pid.h：extern 声明 a_m
  -> main.c：包含 pid.h 后读取 a_m
  -> OLED：显示当前 PWM 输出
```

## OLED 显示 `a_m`

```c
sprintf(text, "PWM:%.0f    ", a_m);
OLED_ShowString(8, 6, (u8 *)text, 16);
```

OLED 会以整数形式显示当前 PID 输出，便于观察正常稳态、加载和卸载瞬间的 PWM 变化。

## 实验现象与数据

### 正常稳定状态

| 项目 | 实验结果 |
|---|---:|
| 目标速度 | 约 `0.50 r/s` |
| 空载稳定 PWM | 约 `744` |

在当前供电、电机和机械负载条件下，电机维持约 `0.50 r/s` 时，`a_m` 稳定在约 744。

### 加阻力后

对输出轴施加阻力后，实验过程为：

```text
电机速度下降
  -> 正误差增大
  -> PID 提高 PWM
  -> a_m 上升到 1000
```

松手卸载后，观察到最高速度约为 `0.8x r/s`。

## 2 秒与 10 秒对照实验

为了判断输出到达上限后是否仍在继续积累积分，进行了两组对照实验：

| 实验 | 饱和状态 | 松手后的最高速度 |
|---|---|---:|
| 实验一 | `a_m` 达到 1000 后继续保持约 2 秒 | 约 `0.8x r/s` |
| 实验二 | `a_m` 达到 1000 后继续保持约 10 秒 | 约 `0.8x r/s` |

虽然第二次保持在输出上限的时间明显更长，但两次松手后的最高速度都在 `0.8x r/s` 左右，没有因为饱和时间从约 2 秒增加到约 10 秒而出现更严重的峰值。

这说明在 `a_m` 达到上限且误差仍要求继续增大输出时，积分项已经停止向饱和方向累积，条件积分抗饱和逻辑发挥了作用。

## 抗积分饱和与超调的区别

抗积分饱和成功不代表完全消除超调。

本次实验中：

```text
松手瞬间 PWM = 1000
空载稳态 PWM ≈ 744
```

负载突然消失时，PWM 还没有来得及从 1000 降回约 744，因此电机获得的驱动力暂时高于空载维持目标速度所需的驱动力，仍会出现短暂超调。

条件积分解决的是“输出到达极限后，积分继续无效累积”的问题，不会消除系统惯性、控制周期延迟以及卸载瞬间输出过高造成的全部超调。

## 故障排查记录

### 故障现象

- 电机能够正常转动
- `speed` 接近 0
- `a_m` 一直保持 1000
- TIM3 中断计数正常递增
- `TIM4->CNT` 只在 0 和 1 之间跳动

### 排查思路

TIM3 中断能够正常运行，说明周期测速任务仍在执行；但 TIM4 的编码器计数几乎没有变化，说明问题更可能出现在编码器信号或连接链路，而不是 PID 公式本身。

```text
确认 TIM3 中断运行
  -> 观察 TIM4->CNT
  -> 发现编码器计数异常
  -> 检查编码器 A、B 两相信号与接线
  -> 发现 B 相自制连接线散架
```

### 故障原因与修复结果

最终发现编码器 B 相的自制连接线散架，B 相信号无法可靠送入 STM32，导致 TIM4 无法正常进行两相正交解码。

修复连接线后，手动转动输出轴一圈，计数约为 1560，编码器反馈恢复正常。

这次排查说明：闭环控制异常不一定来自 PID 参数或算法。反馈传感器、接线和信号完整性出现问题时，PID 会根据错误的反馈继续提高输出，表现为 `speed` 异常且 `a_m` 长时间保持上限。

## 今日总结

今天完成了 PID 条件积分抗饱和实验，并通过约 2 秒和约 10 秒的饱和时间对照验证了算法效果。两次卸载后的最高速度都在 `0.8x r/s` 左右，说明达到输出上限后继续保持更久，没有造成更严重的积分累积。

同时也明确了抗积分饱和与消除超调不是同一件事：松手瞬间 PWM 仍为 1000，而空载稳态只需要约 744，因此仍会出现短暂超调。

本次实验还完成了编码器反馈故障的定位。通过依次检查 TIM3 中断、TIM4 计数以及 A/B 相接线，最终发现并修复了编码器 B 相连接线故障。修复后，输出轴一圈计数约为 1560。
