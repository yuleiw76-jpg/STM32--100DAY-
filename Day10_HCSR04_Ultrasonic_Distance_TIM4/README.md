# Day10 - HC-SR04 超声波测距与 TIM4 定时器计时

## 今日学习目标

- 学会使用 HC-SR04 超声波测距模块
- 理解 TRIG 和 ECHO 的信号方向与作用
- 使用 TIM4 测量 ECHO 高电平持续时间
- 理解 16 位定时器溢出与 `overcount`
- 掌握距离换算公式
- 使用 OLED 显示测量结果
- 理解均值滤波、阻塞等待与超时保护

## 实验功能

本实验由 STM32 向 HC-SR04 发送触发信号，再测量 ECHO 高电平脉宽。程序根据 TIM4 的当前计数值和溢出次数计算声波往返时间，连续测量 5 次取平均值，最后把距离转换成字符串并显示在 OLED 上。

```text
TRIG 输出触发脉冲
-> HC-SR04 发射超声波
-> ECHO 输出高电平脉冲
-> TIM4 统计脉宽
-> overcount 扩展计数范围
-> 换算距离
-> 5 次均值滤波
-> OLED 显示
```

## 硬件连接

以下连接根据本工程实际源码整理：

| 模块信号 | STM32 引脚 | GPIO 模式 | 作用 |
|---|---|---|---|
| HC-SR04 TRIG | PA11 | 推挽输出 `GPIO_Mode_Out_PP` | STM32 输出触发脉冲 |
| HC-SR04 ECHO | PA12 | 浮空输入 `GPIO_Mode_IN_FLOATING` | STM32 读取回波脉冲 |
| OLED SCL | PA6 | 推挽输出 | GPIO 模拟 I2C 时钟线 |
| OLED SDA | PA7 | 推挽输出 | GPIO 模拟 I2C 数据线 |

> `oled.c` 和 `oled.h` 顶部的旧注释仍写着 PD6/PD7，但实际宏定义和初始化代码使用的是 PA6/PA7，应以执行代码为准。

## HC-SR04 工作原理

TRIG 是 STM32 的输出、HC-SR04 的输入；ECHO 是 HC-SR04 的输出、STM32 的输入。

一次测量的基本过程：

1. STM32 将 PA11 拉高。
2. 保持 20 us 后将 PA11 拉低，满足 HC-SR04 至少约 10 us 的触发要求。
3. HC-SR04 发射 8 个 40 kHz 超声波脉冲。
4. 程序等待 PA12 上的 ECHO 信号由低变高。
5. ECHO 变高后清零计数并启动 TIM4。
6. 程序等待 ECHO 由高变低。
7. ECHO 结束后停止 TIM4，读取脉宽对应的总计数。

触发代码：

```c
GPIO_SetBits(GPIOA, GPIO_Pin_11);
delay_us(20);
GPIO_ResetBits(GPIOA, GPIO_Pin_11);
```

## 距离计算

ECHO 高电平持续时间对应超声波从传感器到障碍物再返回传感器的往返传播时间，因此单程距离为：

```text
距离 = 高电平持续时间 * 声速 / 2
```

代码使用声速 `340 m/s`，再乘以 `100` 将米转换为厘米：

```text
距离(cm) = 总计数 * (1 / 72000000) * 340 * 100 / 2
```

除以 2 是因为超声波经历了：

```text
传感器 -> 障碍物 -> 传感器
```

程序测得的是往返路程，而目标距离是单程路程。

## TIM4 配置

主程序中的初始化调用为：

```c
TIM4_Init(65536 - 1, 0);
```

实际参数：

| 参数 | 实际值 | 含义 |
|---|---:|---|
| PSC | 0 | 实际分频系数为 `PSC + 1 = 1`，即不分频 |
| ARR | 65535 | 计数范围为 0 到 65535，共 65536 个计数 |
| 计数模式 | 向上计数 | `TIM_CounterMode_Up` |
| 更新中断 | 已启用 | 每次溢出进入 TIM4 更新中断 |

本工程的距离公式按 TIM4 时钟为 72 MHz 计算，因此：

```text
单次计数时间 = 1 / 72000000 s
             = 13.89 ns
```

一个完整 16 位计数周期约为：

```text
65536 / 72000000 s = 910.22 us
```

## 两个阻塞式等待循环

等待 ECHO 上升沿：

```c
while(GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_12) == RESET);
```

只要 ECHO 仍为低电平，CPU 就一直等待；当 ECHO 变为高电平时退出循环。

等待 ECHO 下降沿：

```c
while(GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_12) == SET);
```

只要 ECHO 保持高电平，CPU 就一直等待，同时 TIM4 在硬件中计数；当 ECHO 变回低电平时退出循环并停止计时。

这两个循环不是设计上的永久死循环，但都属于阻塞式等待。如果传感器断线、目标超出范围或 ECHO 电平异常，程序可能永远无法退出。

## `overcount` 与定时器溢出

`overcount` 在 `tim.c` 中定义：

```c
int overcount = 0;
```

TIM4 更新中断中执行：

```c
overcount++;
```

TIM4 是 16 位定时器。每当 CNT 从 65535 溢出回 0，`overcount` 加 1，用于记录已经完成的整轮计数。

`hcsr04.c` 通过下面的声明访问同一变量：

```c
extern int overcount;
```

`extern` 只是跨源文件声明，不会重新定义变量或再次分配内存。

## 总计数公式

理论上，总计数应为：

```text
总计数 = 当前 CNT + overcount * 65536
```

源码先通过下面的调用取得当前 CNT：

```c
tim = TIM_GetCounter(TIM4);
```

因此预期写法应等价于：

```c
tim + overcount * 65536
```

每次测量开始和结束时，代码都会清零：

```c
overcount = 0;
TIM4->CNT = 0;
```

这样下一次测量不会叠加上一次的数据。

## 实际计时流程

```text
PA11 输出 20 us 高电平
-> 等待 ECHO 上升沿
-> overcount 清零
-> TIM4 CNT 清零
-> 启动 TIM4
-> 等待 ECHO 下降沿
-> 停止 TIM4
-> 读取当前 CNT
-> 合并溢出次数
-> 换算距离
```

## 5 次均值滤波

`Senor_Using()` 使用 `while(i != 5)` 连续完成 5 次测量：

```c
sum = length + sum;
/* ...共测量 5 次... */
length = sum / 5;
```

每次测量之间延时 100 ms。取平均可以减小随机波动，让 OLED 上的结果更稳定，但均值滤波对特别离谱的异常值不够敏感，后续可以尝试中值滤波或去极值平均。

## OLED 显示

主程序的显示流程为：

```c
a = Senor_Using();
sprintf(text, "%.2fcm       ", a);
OLED_ShowString(0, 0, (u8 *)text, 16);
```

`Senor_Using()` 返回厘米单位的距离，`sprintf()` 将浮点数格式化为保留两位小数的字符串，`OLED_ShowString()` 再把结果显示在 OLED 左上角。

## 代码检查与风险

### 1. 溢出轮次常量写成了 `63336`

源码当前实际写法是：

```c
length = 100 * (tim + overcount * 63336)
       * (1.0 / 72000000.0) * 340.0 / 2;
```

但 ARR 为 65535 时，每个完整周期包含 65536 个计数，理论上应使用 `65536`。`63336` 很可能是笔误，会低估每个完整溢出周期对应的时间；本上传版本为了保持实验原貌，没有修改源码。

### 2. 两个 ECHO 等待循环没有超时保护

当前代码没有超时退出机制：

```c
while(ECHO == RESET);
while(ECHO == SET);
```

若 ECHO 始终不变，主程序会卡住。后续应根据最大允许回波时间增加硬件定时超时，并在测量失败时返回错误状态。

### 3. OLED 字符缓冲区过小

`main.c` 定义：

```c
char text[10];
```

但格式字符串 `"%.2fcm       "` 即使显示 `0.00cm` 也需要至少 14 字节（包含结尾的 `\0`），存在越界写入风险。后续应增大缓冲区并优先使用带长度限制的格式化函数；本次未修改原始代码。

### 4. 中断共享变量未声明为 `volatile`

`overcount` 在 TIM4 中断中修改、在普通函数中读取。更稳妥的写法通常会使用 `volatile`，防止编译器把它当作不会异步变化的普通变量。本次只记录风险，不改动实验源码。

### 5. 部分注释与代码不一致

- `hcsr04.c` 的个别注释仍写着 TIM2，实际代码使用 TIM4。
- OLED 文件顶部注释写 PD6/PD7，实际代码使用 PA6/PA7。
- 测距函数名为 `Senor_Using()`，保留了原工程拼写。

判断工程行为时应以实际代码为准。
