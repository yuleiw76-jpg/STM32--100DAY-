今日目标

今天学习 PWM 控制舵机，理解 PWM 的基本原理，以及如何通过 STM32 定时器输出 PWM 波形来控制舵机角度。

本次实验目标是：

按下 KEY1，舵机转到 0°；

今天学了
① PWM 的基本概念

PWM 是脉冲宽度调制，本质上是一种由高电平和低电平组成的数字波形。

通过改变一个周期内高电平持续的时间，就可以改变占空比。

占空比可以理解为：

占空比 = 高电平时间 / 总周期

需要注意的是：

对 LED、电机这类器件来说，占空比常常影响平均输出效果；
但对普通模拟舵机来说，它更关心的是每个周期内的高电平时间，而不是单纯的平均电压。

② PWM 和定时器的关系

PWM 不是 STM32 的一个独立外设，而是定时器 TIM 的一种输出功能。

定时器通过 PSC 和 ARR 决定 PWM 的周期和频率：

PSC：预分频系数
ARR：自动重装载值
CCR：捕获/比较寄存器，用来控制高电平时间

可以简单理解为：

PWM 频率由 PSC 和 ARR 决定；
PWM 占空比主要由 CCR 决定。
③ 舵机 PWM 控制规律

普通 180° 模拟舵机通常使用周期为 20ms，也就是 50Hz 的 PWM 信号控制。

常见对应关系是：

0.5ms 高电平 → 0°
1.5ms 高电平 → 90°
2.5ms 高电平 → 180°

本实验中，如果定时器计数频率为 1MHz，也就是 1us 计数一次，那么：

CCR = 500  → 高电平 500us  → 0.5ms  → 0°
CCR = 1500 → 高电平 1500us → 1.5ms  → 90°
CCR = 2500 → 高电平 2500us → 2.5ms  → 180°
④ 为什么使用复用推挽输出

本实验使用 TIM1_CH1 输出 PWM，对应引脚是 PA8。

因为 PWM 波形不是由 GPIO 自己普通输出高低电平，而是由 TIM1 定时器外设自动控制高低电平，所以 PA8 需要配置成：

GPIO_Mode_AF_PP

也就是复用推挽输出模式。

我的理解是：

普通推挽输出：GPIO 自己控制引脚高低电平；
复用推挽输出：GPIO 把引脚控制权交给外设，比如 TIM1 输出 PWM。
⑤ TIM_SetCompare1(TIM1, 500) 的作用

今天学到：

TIM_SetCompare1(TIM1, 500);

这句话的作用是设置 TIM1 通道 1 的 CCR1 值为 500。

也就是说，它修改的是：

TIM1_CCR1

在本实验中，ARR = 20000 - 1，PWM 周期为 20ms。

所以：

CCR = 500
高电平时间 = 500us = 0.5ms
占空比 = 500 / 20000 = 2.5%
舵机大约转到 0°

注意这里是 CCR，不是 RRC。

⑥ PWM 模式和输出极性

PWM 输出还和 PWM 模式、输出极性有关。

在 PWM 模式 1、向上计数、输出极性为高时：

CNT < CCR：输出高电平
CNT >= CCR：输出低电平

如果改变 PWM 模式或者输出极性，实际输出的高低电平逻辑也可能发生变化。

实验情况

今天完成了 PWM 控制舵机实验。

实验现象：

按下 KEY1，舵机转到 0°；
按下 KEY2，舵机转到 180°。

这说明 TIM1_CH1 成功从 PA8 输出 PWM 信号，并且可以通过改变 CCR 值控制舵机角度。

最大收获

今天最大的收获是理解了：

CCR 可以改变 PWM 的高电平时间；
PWM 的高电平时间可以控制普通模拟舵机的角度。

以前我只知道 PWM 可以控制东西，今天开始知道它在 STM32 里是通过定时器输出比较功能实现的。

也理解了：

PSC 和 ARR 决定 PWM 周期；
CCR 决定高电平时间；
PA8 需要配置成复用推挽输出，因为 PWM 是 TIM1 外设输出的。

今天最困惑的是

今天最困惑的是：

ARR、PSC、CCR 三者之间的关系还不够熟；

现在的初步理解是：

普通模拟舵机主要识别的是每个 20ms 周期内的高电平时间。
比如 0.5ms、1.5ms、2.5ms 分别对应不同角度。
在 PWM 周期固定为 20ms 的情况下，改变 CCR 就是在改变高电平时间，因此可以控制舵机角度。
