# Day 17：P、I、D 单项作用实验与 PI/PID 对比

## 今日学习目标

- 在同一套增量式 PID 工程中分别观察 P、I、D 的原始作用。
- 理解比例、积分、微分三项对启动、稳态误差、振荡和超调的影响。
- 对比 PI 与完整 PID 在受阻后卸载时的动态表现。
- 恢复最终参数 `P = 100`、`I = 30`、`D = 5`、`aim_v = 0.5`。
- 保留条件积分抗饱和与 PWM 最终限幅，暂时关闭 Day16 的输出变化率限制。

## 实验环境和基础条件

- STM32F10x 标准外设库工程，使用 Keil MDK 编译。
- TIM4 使用编码器模式读取电机编码器，输出轴一圈约为 1560 个计数。
- TIM3 周期性计算当前转速 `speed`。
- TIM2 周期性调用 `pid_control(speed)`，并通过 TIM1 PWM 驱动电机。
- OLED 显示当前速度 `speed` 和实际 PWM 输出 `a_m`。
- 目标速度统一设置为 `0.50 r/s`，PWM 输出范围为 `0～1000`。

## 为什么暂时关闭输出变化率限制

Day16 使用了上升 `+30`、下降 `-200` 的非对称输出变化率限制。它可以让启动更平滑并减小卸载后的超调，但也会改变 PID 原始输出到达电机的过程。

Day17 的目标是分别观察 P、I、D 本身造成的现象，因此关闭变化率限制，直接执行：

```c
a_m += off_cal;
```

本工程不使用：

```c
new_output = a_m + off_cal;
```

也没有重新加入每周期上升 `+30`、下降 `-200` 的限制。这样实验中看到的启动、稳态误差、振荡和超调主要来自当前 P、I、D 参数，而不是额外的输出整形。

## 增量式 PID 当前核心代码

```c
float pid_control(float speed)
{
    err_last_last = err_last;
    err_last = err;
    err = aim_v - speed;

    off_cal = P * (err - err_last)
            + D * (err - 2.0f * err_last + err_last_last);

    /* Conditional integration anti-windup. */
    if (!((a_m >= 1000.0f && err > 0.0f) ||
          (a_m <= 0.0f && err < 0.0f)))
    {
        off_cal += I * err;
    }

    /* Incremental PID output update without Day16 slew-rate limiting. */
    a_m += off_cal;

    /* PWM output limit. */
    if (a_m > 1000.0f)
    {
        a_m = 1000.0f;
    }
    else if (a_m < 0.0f)
    {
        a_m = 0.0f;
    }

    return a_m;
}
```

当前最终参数为：

```c
static float P = 100.0f;
static float I = 30.0f;
static float D = 5.0f;

float a_m = 0.0f;
float aim_v = 0.5f;
```

本工程使用 `err = aim_v - speed`，D 项按已经验证的实际源码保留为 `+ D * (...)`，整理时没有改变其符号或控制含义。

## P、I、D 三项的直观含义

- **P：现在差多少。** 当前误差越大，比例项给出的纠偏越强，响应直接且迅速。
- **I：过去累计差了多少。** 只要误差持续存在，积分项就继续累加，用于消除长期稳态误差。
- **D：误差变化得有多快。** 微分项关注误差变化趋势，对变化过快的过程提供一定抑制。

简要总结：

- P 响应快，但可能存在稳态误差，过大会振荡。
- I 可以消除稳态误差，但容易累积并造成超调。
- D 可以抑制变化过快，但不能单独维持目标。

完整 PID 的分工是：P 负责及时纠偏，I 负责消除长期误差，D 负责抑制超调和振荡。

## 纯 P 实验

纯 P 实验统一设置：

```c
aim_v = 0.5f;
I = 0.0f;
D = 0.0f;
```

| P 值 | PWM / 速度表现 | 现象 |
|---:|---|---|
| 100 | PWM 约 50，速度为 0 | 电机不启动 |
| 500 | PWM 约 285，速度为 0 | 电机不启动 |
| 1000 | PWM 约 571，速度为 0 | 电机不启动 |
| 1500 | 速度约 `0.17～0.18 r/s`，PWM 约 `560～580` | 可以慢速转动，但存在明显稳态误差 |
| 2500 | PWM 约 200 多到 900 多，速度约 `0.2x～0.4 r/s` | 电机明显一快一慢，出现振荡 |

实验表明：

- P 太小时输出不足，无法跨过电机启动死区。
- P 增大后可以启动，但纯 P 仍存在稳态误差。
- P 过大时容易振荡。
- 电机一快一慢、PWM 大幅上下变化，就是振荡在实物系统中的表现。

## 纯 I 实验

参数：

```c
P = 0.0f;
I = 30.0f;
D = 0.0f;
```

启动时电机先不转，但误差持续存在，PWM 会在积分作用下不断累加。PWM 超过电机启动死区后，电机开始慢慢转动，最终速度可以稳定在约 `0.50 r/s`，稳定 PWM 约为 `747`。

受阻实验中：

- 按住电机后，PWM 很快增加到 1000。
- 松手后的最高速度约为 `0.90 r/s`。
- 恢复到目标速度附近约需 `2～3 s`。

纯 I 能够消除稳态误差，因为它会持续累加长期存在的误差；但持续受阻时会形成较大的输出积累，松手后容易产生明显超调。当前工程的条件积分抗饱和会在输出达到边界且误差继续把输出推向边界时停止积分，但到达上限前已经形成的高输出仍然会影响卸载瞬间。

## 纯 D 实验

参数：

```c
P = 0.0f;
I = 0.0f;
D = 5.0f;
```

实验结果：PWM 约为 50，速度为 0；手动拨动电机轴后，电机也无法维持转动。

这是因为 D 项只响应误差变化趋势。上电瞬间误差发生变化时会产生短暂输出；当误差稳定后，D 项输出接近零。因此 D 项不能单独让系统达到或维持目标速度。

## PI 和 PID 对照实验

### PI：`P = 100`、`I = 30`、`D = 0`

- 启动到 `0.50 r/s`：约 12 s。
- 松手后最高速度：约 `0.88 r/s`。
- 恢复到目标附近：约 8 s。

### PID：`P = 100`、`I = 30`、`D = 5`

- 启动到 `0.50 r/s`：约 12 s。
- 松手后最高速度：约 `0.82 r/s`。
- 恢复到目标附近：约 7 s。

| 控制方式 | 启动时间 | 松手最高速度 | 恢复时间 |
|---|---:|---:|---:|
| PI：100/30/0 | 约 12 s | 约 `0.88 r/s` | 约 8 s |
| PID：100/30/5 | 约 12 s | 约 `0.82 r/s` | 约 7 s |

## 实验分析

- 加入 D 后，启动时间基本不变，说明当前 `D = 5` 没有明显改变慢速启动过程。
- 卸载最高速度由约 `0.88 r/s` 降到约 `0.82 r/s`。
- 恢复时间由约 8 s 缩短到约 7 s。
- D 项对快速变化过程具有一定抑制作用，相当于给系统增加了一定阻尼。
- `D = 5` 带来的改善不算巨大，但最高速度和恢复时间都朝更稳定的方向变化，趋势明确。

## 今日总结

通过单独启用 P、I、D，我把课本中的三个抽象项对应到了真实电机现象：P 不足时跨不过启动死区，P 过大会让 PWM 和转速振荡；I 能把长期误差积累成足够的输出并消除稳态误差，但受阻后容易超调；D 不能单独维持速度，却能在 PI 基础上对快速变化增加阻尼。

最终恢复 `P = 100`、`I = 30`、`D = 5` 后，PID 与 PI 的启动时间都约为 12 s，但卸载峰值和恢复时间均有所改善。这次实验说明，完整 PID 的价值不是让某一项独自完成全部控制，而是让三项分别处理及时纠偏、长期误差和变化趋势。
