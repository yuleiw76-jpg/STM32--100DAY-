
## 实验基础与最终参数

目标速度为：

```c
float aim_v = 0.5f;
```

Day18 最终保留：

```c
static float P = 100.0f;
static float I = 30.0f;
static float D = 5.0f;
```

工程化限制包括：

- PWM 每个控制周期最多增加 30。
- PWM 每个控制周期最多减少 200。
- PWM 最终限制在 `0～1000`。
- 输出达到边界且误差继续把输出推向边界时，停止积分。

## 当前 PID 核心代码

```c
float pid_control(float speed)
{
    float new_output;

    err_last_last = err_last;
    err_last = err;
    err = aim_v - speed;

    off_cal = P * (err - err_last)
            + D * (err - 2.0f * err_last + err_last_last);

    /* Stop integration only when it would drive the output farther into saturation. */
    if (!((a_m >= 1000.0f && err > 0.0f) ||
          (a_m <= 0.0f && err < 0.0f)))
    {
        off_cal += I * err;
    }

    new_output = a_m + off_cal;

    if (new_output - a_m > 30.0f)
    {
        new_output = a_m + 30.0f;
    }
    else if (new_output - a_m < -200.0f)
    {
        new_output = a_m - 200.0f;
    }

    a_m = new_output;

    if (a_m > 1000.0f)
    {
        a_m = 1000.0f;
    }
    else if (a_m < 0.0f)
    {
        a_m = 0.0f;
    }

    return a_m;
}
```

本工程定义 `err = aim_v - speed`，微分增量按已经验证的源码保留为 `+ D * (...)`。

## PID 参数整定思路

- **P：当前误差有多大。** 主要影响纠偏强度和响应速度。
- **I：误差已经持续了多久。** 用于消除长期存在的稳态误差。
- **D：误差变化得有多快。** 用于抑制快速变化和超调。

调参时需要同时观察：启动时间、稳态精度、卸载超调、恢复时间、PWM 波动和电机振荡。

## P 参数对比

保持 `I = 30`、`D = 5`，分别比较 `P = 50、100、200`：

| P | 启动表现 | 超调表现 | 系统特点 |
|---:|---|---|---|
| 50 | 启动较慢 | 超调较小 | 控制作用偏弱，响应较慢 |
| 100 | 启动速度适中 | 超调适中 | 综合表现较均衡 |
| 200 | 启动更快 | 超调和波动增大 | 纠偏更强，但稳定性下降 |

增大 P 会让控制器更积极地纠正当前误差，但过大的 P 容易形成“PWM 增加过多、速度上升过快、PWM 再快速下降”的循环，从而产生超调或振荡。

## I 参数对比

保持 `P = 100`、`D = 5`，分别比较 `I = 10、30、60`：

| I | 达到目标速度 | 超调表现 | 系统特点 |
|---:|---|---|---|
| 10 | 较慢 | 较小 | 积分较弱，消除误差较慢 |
| 30 | 速度适中 | 适中 | 稳态精度和超调较均衡 |
| 60 | 更快接近目标 | 超调明显增大 | 积分累积更快，更容易过量补偿 |

增大 I 可以更快消除稳态误差，但电机受阻时误差会持续存在，较大的积分作用会更快把 PWM 推向高值，负载突然消失后更容易超调。

## D 参数对比

保持 `P = 100`、`I = 30`，分别比较 `D = 0、5、20`：

| D | 启动表现 | 卸载超调 | 系统特点 |
|---:|---|---|---|
| 0 | 启动正常 | 超调较明显 | 缺少对快速变化的抑制 |
| 5 | 启动变化不明显 | 超调有所降低 | 系统阻尼得到改善 |
| 20 | 响应更加保守 | 超调进一步受抑制 | 过大时系统可能变钝，并放大测速波动 |

D 项对误差变化趋势作出反应。适当增大 D 可以在负载突然消失、速度快速上升时更早降低输出；D 过大则会让控制器对测速噪声和小幅波动过于敏感。

## 参数变化规律

| 参数增大 | 主要收益 | 主要风险 |
|---|---|---|
| P | 响应更快、纠偏更强 | 超调增大，过大时振荡 |
| I | 更快消除稳态误差 | 积分累积和卸载超调增大 |
| D | 抑制快速变化、增加阻尼 | 响应变钝，对测速噪声更敏感 |

## 常见现象与调参方向

| 现象 | 优先检查或调整方向 |
|---|---|
| 启动或响应太慢 | 适当增大 P 或 I，但一次不要改动过大 |
| 长期达不到目标 | 检查输出能力，并适当增大 I |
| 卸载后超调过大 | 适当减小 P/I，适当增大 D，检查 PWM 下降速度 |
| 电机明显一快一慢 | 可能发生振荡，减小 P/I 并检查测速数据 |
| PWM 长时间停在 1000 | 检查堵转、目标是否过高及抗积分饱和逻辑 |
